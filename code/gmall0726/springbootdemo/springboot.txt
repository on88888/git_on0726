一、路径

类比linux文件系统中的路径

        相对路径：  不以/开头。如果想获取相对路径的绝对路径，必须知道当前路径相对的父路径
                        举例：  abc

                        举例：  abc 在 /opt/module/目录下

                        此时它的绝对路径：   /opt/module/ abc

        绝对路径：  以/开头
                        举例： /abc



    在项目中都建议编写绝对路径（规范）！

    /如何解析：
            ①/ 写在.java文件中，它会被解析为  http://主机:端口/项目路径/

            ②/ 写在html文件中，它会被解析为  http://主机:端口/

                    举例：<a href="/abc" >
                            http://localhost:63342/abc

                    在html中如果写绝对路径，需要手动拼接项目的根路径(项目名)！



二、响应数据而不是响应页面

        要响应的数据分类：
                字面量：   可以从字面上就知道变量值的变量.
                                常见的基本数据类型和包装类都是字面量！

                            int i = 0
                            String s = "hahaha"

                            直接将字面量放入请求体！不做任何处理！

                非字面量：
                            自己定义的Bean都是非字面量
                                Employee e = new Employee()

                                springboot内置了jackson，会将对象转换为JSON字符串！



        响应的方式:
                将响应的数据return即可。在return 时，需要告诉springboot，当前响应的内容不是一个路径，而是一份数据！
                需要在方法上标注@ResponseBody


三、接收浏览器发送的数据

            浏览器发送的数据：
                上传的数据流（不讨论）： 上传一个文件到网盘，和浏览器建议数据流的连接，获取请求体中上传的内容

                浏览器发送参数到服务器端：

                        发送参数的格式：     参数名 = 参数值

                        常见发送参数的请求方式：
                                GET：   url?参数名=参数值&参数名=参数值
                                        浏览器的地址栏和<a>都只能发送GET


                                POST：   url
                                        请求体中
                                            参数名=参数值



                服务器端如何接收：
                        在处理请求的方法的形参位置声明和请求参数属性名一致的形参即可！
